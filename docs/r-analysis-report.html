RETIRE  Components to Retire

These components use deprecated APIs, contain critical bugs, or present security vulnerabilities that make them unsuitable to keep as-is.

SpringSecurityConfig — WebSecurityConfigurerAdapter

src/main/java/com/example/moodsense/demo/config/SpringSecurityConfig.java

Why retire: Extends the deprecated WebSecurityConfigurerAdapter (removed in Spring Security 6). Contains hardcoded credentials (user1/pwd1, user2/pwd2), disables CSRF globally, and exposes the H2 console without authentication. The entire class should be replaced with a modern SecurityFilterChain @Bean approach, externalizing credentials and removing H2 console access in non-dev profiles.

Distance Calculation in getBestLocationFromCoordinate

src/main/java/com/example/moodsense/demo/service/MoodServiceImplementation.java (lines 43-45)

Why retire: The Euclidean distance formula contains a confirmed bug — the second squared term uses location.getLatitude() instead of location.getLongitude(). This produces incorrect nearest-location results. The buggy logic must be discarded and rewritten with the correct formula, ideally extracted into a dedicated spatial utility.

CrudRepository<Mood, String> ID Type

src/main/java/com/example/moodsense/demo/repositary/MoodRepository.java

Why retire: The repository declares CrudRepository<Mood, String> while the entity Mood uses @Id private int id. This type mismatch is fundamentally broken for any ID-based operations (findById, deleteById). Must be changed to JpaRepository<Mood, Integer>.

Package "repositary" (misspelling)

src/main/java/com/example/moodsense/demo/repositary/

Why retire: The package name is misspelled. It should be renamed to repository along with all import statements referencing it.

  • Return ResponseEntity<?> consistently with proper HTTP status codes (201 for POST)

  • Add a consistent response envelope structure

MoodServiceImplementation — N+1 Queries & Null Safety

src/main/java/com/example/moodsense/demo/service/MoodServiceImplementation.java
