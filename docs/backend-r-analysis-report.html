Logic correctness – distance calculation bug biz_001
In MoodServiceImplementation.getBestLocationFromCoordinate (line 45), the Euclidean distance formula is incorrect. The second squared term computes `location.getLatitude() - coordinate.getLongitude()` instead of `location.getLongitude() - coordinate.getLongitude()`. This means the distance is calculated using latitude twice (once against the coordinate's latitude and once against the coordinate's longitude), completely ignoring the location's longitude. This will select the wrong nearest location in many cases.
